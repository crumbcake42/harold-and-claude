# sca-tracker — Frontend

Vite + React + TypeScript SPA. The backend API client is generated from the
committed OpenAPI contract.

## Build & run

- `pnpm dev` — dev server
- `pnpm test` — vitest (run once)
- `pnpm lint` / `pnpm lint:fix` — ESLint (+ Prettier)
- `pnpm typecheck` — `tsc -b`
- `pnpm build` — production build
- `pnpm gen-api` — regenerate the API client from `../contracts/openapi.json`
  (or `just gen-openapi` to re-export the schema from the backend first)

## Conventions

**Read `src/PATTERNS.md` before building any feature.** It owns the four-layer
architecture, the per-feature API-wrapper layer, routing, forms, and testing.

Non-obvious points:

- **Import alias is `@/` only** (→ `src/`). No other alias.
- **Backend payload types** come from `@/api/generated/types.gen` — never
  hand-rolled. A missing or `unknown` type is a backend bug: fix the FastAPI
  response model and regenerate.
- **`src/api/generated/` is generated** — committed, regenerated by
  `pnpm gen-api`, never hand-edited; CI fails on drift. Hand-written API code
  goes in `src/api/` *outside* `generated/` (e.g. `src/api/configure.ts`).
- **Auth is cookie-session** (`httpOnly` cookie, `credentials: 'include'`), not
  token-based — no token store, no 401 interceptor. See ADR-0063.
- **UI primitives** are shadcn/ui in the `radix-lyra` style (`src/components/ui/`,
  vendored). No `form` component — use the `Field` family with react-hook-form
  + Zod and `standardSchemaResolver`.
- **Icons:** Phosphor, `*Icon`-suffixed (`SignOutIcon`).
- **React event types:** use `SubmitEvent` / `ChangeEvent` — `FormEvent` is
  deprecated in `@types/react` 19.
